# Implementation Plan: Fallback Models

**Branch**: `008-model-fallback` | **Date**: 2026-03-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/008-model-fallback/spec.md`

## Summary

Add virtual model entries to `model_list` config. A virtual entry carries a `fallbacks` array of other model names instead of a direct `provider`/`upstream_model`. At startup, virtual chains are flattened and validated (cycle detection, unknown references). At runtime, both proxy handlers iterate the chain and retry on 5xx, 429, or network error, stopping at the first success or logging exhaustion. A new `fallback_attempts` column on `request_logs` records how many provider dispatches were made per request.

## Phase 0: Research

See [research.md](research.md). All decisions resolved; no NEEDS CLARIFICATION items remain.

Key decisions:
- **Chain representation**: `ModelMapping.Fallbacks []string`; mutually exclusive with `Provider`/`UpstreamModel`
- **Resolution**: Eager flatten at `Config.Load`; `FindModel` returns `[]ModelMapping`
- **Fallback triggers**: `err != nil` from `SendRequest`, `status >= 500`, `status == 429`
- **Streaming boundary**: Check `provResp.StatusCode` after `SendRequest`, before `handleStreaming`; close stream on fallback
- **Logging**: `fallback_attempts INT NOT NULL DEFAULT 1`; `logRequest` gains `attemptCount int` param

### `isFallbackStatus` helper

```go
func isFallbackStatus(code int) bool {
    return code >= 500 || code == 429
}
```

Lives in `internal/proxy/` (unexported, used by both handlers).

### Handler retry loop (both `Handler` and `OpenAIHandler`)

Pseudocode for the dispatch section of `ServeHTTP`:

```
chain, err = cfg.FindModel(reqBody.Model)   // []ModelMapping
if err ‚Üí 400 unknown model

for i, mapping := range chain:
    prov = providers[mapping.Provider]
    provResp, err = prov.SendRequest(ctx, req_with_upstream_model)

    if err != nil || isFallbackStatus(provResp.StatusCode):
        if err == nil: provResp.Stream.Close()   // don't leak open connection
        attempts++
        if i == len(chain)-1: ‚Üí 503 all entries exhausted (log attempts)
        continue

    // success
    if streaming:
        handleStreaming(... attempts+1)
    else:
        handleNonStreaming(... attempts+1)
    return

// not reached
```

`logRequest` gains a final `attemptCount int` parameter; callers pass `i+1`.

### Config load additions (in `Config.Load`)

```
1. For each ModelMapping in cfg.ModelList:
   - Validate mutually exclusive fields
2. Build name‚Üímapping index
3. Cycle detection: DFS over virtual entries
4. Flatten: for each virtual model, recursively expand fallbacks into []ModelMapping
5. Populate cfg.resolvedChains
```

`FindModel` checks `resolvedChains` first (covers both direct and virtual), then falls back to wildcard logic for direct entries not in the map.
